## 4. 내가 틀렸고 바로잡은 것

> 같은 실수를 반복하지 않기 위해 남긴다.

### 파이썬 `\n` 이 JS·노트북 문자열을 끊음 (3회)

- **노트북** : `print('\n[결측치]')` 가 두 줄로 갈라져 `SyntaxError`
- **index.html** : JS 문자열이 끊겨 `Invalid or unexpected token` → **로그인 버튼조차 동작 안 함**
- **조치** : raw 문자열(`r'''...'''`) 사용 + Write 도구로 파일 작성 + `check_js.py` (5개 페이지 실제 열어 JS 에러·핸들러 확인)

### 확인하지 않은 수치를 씀

- SAMPLE_3 해설에 "해조류양식어업 5.51%" → 실제 **38.22%**, 최저도 아님
- 더 파보니 진짜 원인은 **선적항 충청북도 49척 전부 소형(0.00%)**
- 이 사실을 해설에 넣고 *"49척뿐인 희소 범주의 100% 는 믿으면 안 된다"* 는 교훈까지 추가

### PDF 머리글/바닥글을 없앨 수 있다고 판단

- `@page margin:0` → **Chrome 이 그대로 그림**, 오히려 본문과 겹침
- 문서 제목 비우기 → **그 자리에 URL 이 대신 들어감** (더 나빠짐)
- **결론** : 브라우저 인쇄 설정이라 페이지가 못 건드림 → 안내 모달로 해결

### 글자 검색만 믿고 "URL 없음" 이라 보고

- Microsoft Print to PDF 는 글자를 **도형으로 그려** 검색 불가 (추출 0자)
- 화면으로 렌더링하니 URL 이 명확히 있었음
- **조치** : PyMuPDF 로 실제 렌더링해 확인

### 안내 문구가 실제와 달랐음

- `옵션 더보기` → 실제 **`설정 더보기`**
- `머리글 및 바닥글` → 실제 **`머리글과 바닥글`** (위치도 다름)
- 사용자 캡처 덕분에 발견. 없는 이름을 찾으라고 안내하고 있었음

### 치환 실패가 조용히 넘어감

- `assert` 가 다른 치환 성공에 가려져 통과
- 행에 `data-m` 도 잠금 버튼도 안 들어갔는데 성공으로 보고됨
- **조치** : 생성 결과를 직접 세는 검사 추가 (행 17개 · 버튼 17개 · 점수 열 36칸)
